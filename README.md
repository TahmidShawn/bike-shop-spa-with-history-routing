# E-Ride Bike Shop (SPA)

Live: [https://bike-shop-spa-with-history-routing-4.onrender.com](https://bike-shop-spa-with-history-routing-4.onrender.com)

---

## About this project

This is a simple bike shop website built as a Single Page Application using vanilla JavaScript.

The main focus of this project is not building a full e-commerce system, but to understand how a SPA works without using any frameworks.

I tried to build routing using the History API and handle page navigation without reloading the page.

---

## What I tried to do

- Build a single page application using plain JavaScript
- Learn how client-side routing works
- Use History API for navigation between pages
- Handle page refresh issues using backend support
- Structure the project like a real SPA

---

## What this project has

- Home page
- About section
- Basic ui layout for a bike shop
- Simple routing without page reload
- Loading screen while pages change

---

## What is not included

This project is still in the learning stage, so some features are not added yet:

- No add to cart system
- No payment or checkout
- No user login or authentication
- No full backend e-commerce logic

---

## What I learned

- How SPA routing works without frameworks
- How History API works in the browser
- Why refresh breaks SPA and how to fix it from backend
- How to structure a small frontend project using raw JavaScript

---

## Goal of this project

The goal was to understand how modern frontend frameworks handle routing and page transitions by building it myself using only JavaScript.
